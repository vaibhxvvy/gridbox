## What is this?

**bggen** is a zero-dependency, browser-based tool for generating CSS background patterns. Open the file, pick a pattern from 12 options, adjust size / opacity / thickness / rotation, and copy the CSS directly into your project.

No npm. No build step. No login. No backend. One `.html` file ‚ÄĒ that's the entire app.

## Quick Copy

Four copy modes in the bottom bar dropdown:

```css
/* copy bg ‚ÄĒ just the background colour */
background-color: #0a0a0a;

/* copy pattern ‚ÄĒ just the image layer */
background-image: radial-gradient(...);
background-size: 20px 20px;

/* copy bg + pattern ‚ÄĒ full drop-in CSS */
background-color: #0a0a0a;
background-image: radial-gradient(...);
background-size: 20px 20px;

/* copy as .bg-pattern ‚ÄĒ ready to paste class */
.bg-pattern {
  background-color: #0a0a0a;
  background-image: radial-gradient(...);
  background-size: 20px 20px;
}
```

## Usage in your project

Paste the copied CSS into any selector:

```css
/* Minimal ‚ÄĒ dots on dark */
body {
  background-color: #0a0a0a;
  background-image: radial-gradient(circle, rgba(255,255,255,0.20) 1px, transparent 1px);
  background-size: 20px 20px;
}

/* Section with grid */
.hero {
  background-color: #0d1b2a;
  background-image:
    linear-gradient(rgba(68,136,255,0.30) 1px, transparent 1px),
    linear-gradient(90deg, rgba(68,136,255,0.30) 1px, transparent 1px);
  background-size: 24px 24px;
}

/* With PNG export */
.section {
  background-image: url('bggen-dots-1920x1080.png');
  background-size: cover;
}
```

## Performance

bggen is designed to stay smooth even on rapid slider input:

- **Dirty-flag render loop** ‚ÄĒ rapid slider drags collapse into one canvas draw per frame via `requestAnimationFrame`
- **Active-only thumb updates** ‚ÄĒ while dragging a slider, only the current pattern's thumbnail redraws. All other 11 thumbnails are untouched until you stop
- **Staggered full redraws** ‚ÄĒ on pattern switch, thumbnails redraw one at a time (20ms apart) so the main thread never spikes
- **Debounced thumb scheduling** ‚ÄĒ thumbnail redraws are debounced 280ms after the last input event
